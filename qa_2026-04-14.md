## Findings

### CRIT Blockers for Live Launch

1. **USDC allowance for CTF Exchange = $0** (I-14). User PM Proxy `0x06B579Eac867695baecCdABfA060111BD1ABe32B` holds USDC.e but has not approved the CTF Exchange to spend. Live orders will revert on-chain. User must approve via PM UI before live run. **Do not fix in code.**

### Non-blocking Findings

1. **F-10 risk_guard bypassed in paper mode (behavioral).** `PMTrader.place_order` only calls `self._risk.authorize(...)` inside the `if not self.paper_mode:` branch (src/pmbot/trading/trader.py:305–319). When a HALT_ALL row exists in risk_state, paper orders still execute.
   - Impact for paper mode: none (no real money).
   - Impact for live mode: correct — live path does enforce HALT_ALL.
   - Recommendation (not blocking live launch): move the risk check above the paper_mode split so paper mode mirrors live denial semantics. ≤10 LOC change. Left un-fixed per QA rule ("don't modify gates/scoring/planner"); risk_guard arguably IS a gate.

2. **CLI UX: unhandled exceptions printed as tracebacks** (A-1, B-5). `KeystoreError` and `ValueError` escape the click context into stderr traceback. Error messages are clear; no secrets leak. Nice-to-have: wrap in `click.ClickException`. Non-blocking.

3. **G-11 STARTUP/SHUTDOWN orphan** is benign — there is a parallel long-running daemon (PID 51411) which accounts for the extra STARTUP.

4. **Preflight shows 5 SKIPPED without password**, vs spec's expected 3-4. Extra skips are EOA-resolves and L2-API-creds, both legitimately gated on unlocked keystore. Non-blocking.

## Go / No-Go Recommendation

**NO-GO for `--live`.**

Reasoning (conservative):
- Pre-live blocker #1 (USDC → CTF Exchange allowance = $0) will cause every live order to fail on-chain at settlement. Must be resolved before live launch. Only the human user can approve this via the PM UI.
- All code paths exercised in paper mode behave correctly (except the paper-mode risk_guard gap, which does not affect live safety).
- 218/218 unit tests green; config defaults safe; daemon startup/shutdown clean; preflight passes up to the gated items.

Once the allowance is approved on-chain:
1. Re-run `pmbot preflight --prompt-password` with real password — expect all 12 checks green.
2. Place a single tiny live order via `pmbot test-order --live --confirm-real-trade --qty 1 --price <far-from-mid>` to verify end-to-end signing + settlement.
3. Then proceed to `pmbot run --live --confirm-real-trade`.
